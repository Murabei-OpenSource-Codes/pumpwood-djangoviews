 67class MicroserviceForeignKeyField(serializers.Field):
 68    """
 69    Serializer field for ForeignKey using microservice.
 70
 71    This serializer will fetch object data from a microservice using a
 72    microservice object.
 73
 74    This field makes a call to the loadbalancer, caution when microservice
 75    calls the same microservice of origin or serialization might result on
 76    circular fetching.
 77
 78    MicroserviceForeignKeyField are always set as `required=False` and
 79    `read_only=True`.
 80
 81    Example of usage:
 82    ```python
 83    class MetabaseDashboardSerializer(DynamicFieldsModelSerializer):
 84        pk = serializers.IntegerField(
 85            source='id', allow_null=True, required=False)
 86        model_class = ClassNameField()
 87
 88        # Foreign Key
 89        # Will return the object data associated with updated_by_id at
 90        # updated_by key at the serialized object.
 91        updated_by = MicroserviceForeignKeyField(
 92            model_class="User", source="updated_by_id",
 93            microservice=microservice, display_field='username')
 94    ```
 95    """
 96
 97    microservice: PumpWoodMicroService
 98    """Microservice object that will make retrieve calls for foreign key
 99       object."""
100    model_class: str
101    """String setting the foreign key destiny model_class."""
102    display_field: str
103    """Set a display field to be returned as __display_field__ at foreign key
104       object. This might help frontend rendering dropdown or other
105       visualizations."""
106    fields: List[str]
107    """List of the field that will be returned with foreign key object."""
108
109    def __init__(self, source: str, microservice: PumpWoodMicroService,
110                 model_class: str,  display_field: str = None,
111                 fields: List[str] = None, **kwargs):
112        """
113        __init__.
114
115        Args:
116            source [str]:
117                Source attribute that contains id value associated with
118                a foreign key.
119            microservice [PumpWoodMicroService]:
120                PumpWoodMicroService object that will be used to fetch
121                information of the object associated with foreign key id.
122            model_class [str]:
123                Model class that will be used to request information at
124                with a retrieve.
125            display_field [str]:
126                Field that will be set as `__display_field__` at object
127                dictonary.
128            fields List [str]:
129                List of the fields that should be returned at the object.
130        """
131        self.microservice = microservice
132        self.model_class = model_class
133        self.display_field = display_field
134        self.fields = fields
135
136        # Set as read only and not required, changes on foreign key must be
137        # done using id
138        kwargs['required'] = False
139        kwargs['read_only'] = True
140        super(MicroserviceForeignKeyField, self).__init__(
141            source=source, **kwargs)
142
143    def get_fields_options_key(self):
144        """
145        Return key that will be used on fill options return.
146
147        @private
148        """
149        return self.source
150
151    def bind(self, field_name, parent):
152        """@private."""
153        # In order to enforce a consistent style, we error if a redundant
154        # 'method_name' argument has been used. For example:
155        # my_field = serializer.CharField(source='my_field')
156        if self.field_name is None:
157            self.field_name = field_name
158        else:
159            assert self.field_name != field_name, (
160              "It is redundant to specify field_name when it is the same")
161        super(MicroserviceForeignKeyField, self).bind(field_name, parent)
162
163    def get_attribute(self, obj):
164        """@private."""
165        return obj
166
167    def to_representation(self, obj) -> dict:
168        """
169        Use microservice to get object at serialization.
170
171        Args:
172            obj: Model object to retrieve foreign key associated object.
173        Returns:
174            Return the object associated with foreign key.
175        """
176        self.microservice.login()
177
178        object_pk = getattr(obj, self.source)
179        # Return an empty object if object pk is None
180        if object_pk is None:
181            return {"model_class": self.model_class}
182
183        object_data = self.microservice.retrieve(
184            model_class=self.model_class, pk=object_pk,
185            default_fields=True, fields=self.fields)
186        object_data['__display_field__'] = object_data.get(self.display_field)
187        return object_data
188
189    def to_internal_value(self, data):
190        """
191        Raise error always, does not unserialize objects of this field.
192
193        Raises:
194            NotImplementedError:
195                Always raise NotImplementedError if try to unserialize the
196                object.
197        """
198        raise NotImplementedError(
199            "MicroserviceForeignKeyField are read-only")
200
201    def to_dict(self) -> dict:
202        """
203        Return a dict with values to be used on options end-point.
204
205        Returns:
206            Return a dictonary with information of the field. Keys associated:
207            - **model_class [str]:** Model class associated with foreign key.
208            - **many [str]:** If it will return a list of objects of one.
209                Foreign Key serializer will always return one object.
210            - **display_field [str]:** Display field that will be set to
211                __display_field__ key on serialized object.
212            - **fields [str]:** If set, fields that will be returned by
213                serializer, if not set will return de default list fields.
214            - **object_field [str]:** Name of the object field associated
215                with the foreign key (this field).
216        """
217        return {
218            'model_class': self.model_class,
219            'many': False,
220            'display_field': self.display_field,
221            'fields': self.fields,
222            'object_field': self.field_name}

388class LocalForeignKeyField(serializers.Field):
389    """
390    Serializer field for ForeignKey using database connection.
391
392    This serializer will fetch object data from a database using a
393    connection and converting to object using serializer.
394
395    Serializer can be passed as python object or using a string to the path
396    that will be lazy loaded when first fetch.
397
398    ```python
399    class SerializerPumpwoodMFAMethod(DynamicFieldsModelSerializer):
400        pk = serializers.IntegerField(source='id', allow_null=True)
401        model_class = ClassNameField()
402
403        # ForeignKey
404        user_id = serializers.IntegerField(allow_null=False, required=True)
405        user = LocalForeignKeyField(
406            serializer=(
407                "pumpwood_djangoauth.registration." +
408                "serializers.SerializerUser")
409            display_field="username",
410            fields=["pk", "username"])
411    ```
412    """
413
414    serializer_cache: serializers.ModelSerializer
415    """Serializer can be lazy loaded passing the path to avoid circular import,
416       serializer_cache will cache serializer to remove necessity of importing
417       the serializer at every request."""
418    serializer: Union[str, serializers.ModelSerializer]
419    """Serializer or string to serializer path that will be lazy loaded."""
420    display_field: str
421    """Display field that will be set to `__display_field__` key at the
422       object."""
423    fields: List[str]
424    """Limit the return fields to fields set, if not set will return list
425       default fields."""
426
427    def __init__(self, serializer: Union[str, serializers.ModelSerializer],
428                 display_field: str = None, fields: List[str] = None,
429                 **kwargs):
430        """
431        __init__.
432
433        Args:
434            serializer Union[str, serializers.ModelSerializer]:
435                Serializer can be lazy loaded passing the path to avoid
436                circular import, serializer_cache will cache serializer to
437                remove necessity of importing the serializer at every request.
438            display_field [str]:
439                Display field that will be set to `__display_field__` key at
440                the object.
441            fields List[str]:
442                Limit the return fields to fields set, if not set will return
443                list default fields.
444        """
445        # Avoid circular imports for related models and cache lazzy loaded
446        # serializers
447        self.serializer_cache = None
448        self.serializer = serializer
449        self.display_field = display_field
450        self.fields = fields
451        kwargs['read_only'] = True
452        super(LocalForeignKeyField, self).__init__(**kwargs)
453
454    def get_fields_options_key(self):
455        """
456        Return key that will be used on fill options return.
457
458        @private
459        """
460        model = self.parent.Meta.model
461        parent_field = getattr(model, self.source)
462        return parent_field.field.column
463
464    def to_representation(self, value):
465        if self.serializer_cache is None:
466            if type(self.serializer) is str:
467                self.serializer_cache = _import_function_by_string(
468                    self.serializer)
469            else:
470                self.serializer_cache = self.serializer
471
472        # Return an empty object if object pk is None
473        if value is None:
474            model = self.parent.Meta.model
475            parent_field = getattr(model, self.source)
476            model_class = parent_field.field.related_model.__name__
477            return {"model_class": model_class}
478
479        data = self.serializer_cache(
480            value, many=False, fields=self.fields,
481            default_fields=True).data
482        display_field = data.get(self.display_field, None)
483        data['__display_field__'] = display_field
484        return data
485
486    def to_dict(self) -> dict:
487        """
488        Return a dict with values to be used on options end-point.
489
490        Returns:
491            model_class [str]:
492                Model class associated with foreign key.
493            many [bool]:
494                Return always False, only one object will be returned.
495            display_field [str]:
496                Object display_field will be returned as __display_field__
497                key.
498            object_field [str]:
499                Will return the field associated with this serializer.
500            fields [List[str]]:
501                If not None will restrict the fields returned at object.
502        """
503        model = self.parent.Meta.model
504        parent_field = getattr(model, self.source)
505        model_class = parent_field.field.related_model.__name__
506        return {
507            'model_class': model_class, 'many': False,
508            'display_field': self.display_field,
509            'object_field': self.field_name, 'fields': self.fields}

512class LocalRelatedField(serializers.Field):
513    """
514    Serializer field for related objects using microservice.
515
516    This serializer will fetch a list of objectsfrom a microservice using a
517    microservice object at `list_without_pag` end-point. It will use
518    serilializing object id to query for related object at destiny model
519    class `foreign_key`.
520
521    This field makes a call to the loadbalancer, caution when microservice
522    calls the same microservice of origin or serialization might result on
523    circular fetching.
524
525    MicroserviceRelatedField are always set as `required=False` and
526    `read_only=True`.
527
528    Example of usage:
529    ```python
530    class UserSerializer(DynamicFieldsModelSerializer):
531        pk = serializers.IntegerField(
532            source='id', allow_null=True, required=False)
533        model_class = ClassNameField()
534
535        # Foreign Key
536        # Will return the object data associated with updated_by_id at
537        # updated_by key at the serialized object.
538        api_permission_set = LocalRelatedField(
539            serializer=(
540                "pumpwood_djangoauth.api_permission."
541                "serializers.SerializerPumpwoodPermissionPolicyUserM2M"),
542            order_by=["-id"])
543    """
544
545    def __init__(self, serializer, order_by: List[str] = ["-id"],
546                 fields: List[str] = None, **kwargs):
547        # Avoid circular imports for related models and cache lazzy loaded
548        # serializers
549        self.serializer_cache = None
550        self.serializer = serializer
551        self.order_by = order_by
552        self.fields = fields
553        kwargs['read_only'] = True
554        super(LocalRelatedField, self).__init__(**kwargs)
555
556    def get_fields_options_key(self):
557        """
558        Return key that will be used on fill options return.
559
560        @private
561        """
562        return self.source
563
564    def to_representation(self, value):
565        """
566        Return all related data serialized.
567
568        @private
569        """
570        if self.serializer_cache is None:
571            if type(self.serializer) is str:
572                self.serializer_cache = _import_function_by_string(
573                    self.serializer)
574            else:
575                self.serializer_cache = self.serializer
576        return self.serializer_cache(
577            value.order_by(*self.order_by).all(),
578            many=True, default_fields=True,
579            fields=self.fields).data
580
581    def to_dict(self):
582        """
583        Return a dict with values to be used on options end-point.
584
585        Returns:
586            model_class [str]:
587                Model class associated with related model.
588            many [bool]:
589                Return always True indicating the user will receive a list
590                of objects.
591            pk_field [str]:
592                Pk field associated with origin model class that will be used
593                to query related models at foreign_key.
594            foreign_key [str]:
595                Foreign Key that will be used to fetch realated models using
596                origin model foreign key.
597            order_by [List[str]]:
598                List of fields to be used to order results from realted
599                models.
600            fields [List[str]]:
601                List of fields that will be returned by related model. If not
602                set default list fields from related model will be used.
603        """
604        # Get information from related field
605        model = self.parent.Meta.model
606        parent_field = getattr(model, self.source)
607        foreign_key = parent_field.field.column
608
609        pk_field = parent_field.rel.target_field
610        pk_field_return = \
611            'pk' if pk_field.primary_key else pk_field.column
612
613        model_class = parent_field.rel.related_model.__name__
614        return {
615            'model_class': model_class,
616            'many': True,
617            "pk_field": pk_field_return,
618            'order_by': self.order_by,
619            'fields': self.fields,
620            'foreign_key': foreign_key}

623class DynamicFieldsModelSerializer(serializers.ModelSerializer):
624    """
625    A ModelSerializer that change fields returned on serialization.
626
627    This serializer make possible to change the serialization fields acording
628    to arguments that are passed on serializer instanciation.
629
630    #### Default list fields
631    Attribute `list_fields` set the default fields that will be returned
632    if `default_fields=True` argument is set.
633
634    #### Usage with foreign key
635    Foreign keys can be returned both for many=True and also many=False
636    serializations passing `foreign_key_fields=True` argument.
637
638    #### Usage with related objects
639    Serialization of many objects will not return related models serializers (
640    MicroserviceRelatedField, LocalRelatedField). This behaviour will overide
641    related_fields argumment, even if set as related_fields=True.
642
643    Related Models serizalization is an expensive request at backend, should
644    be used only for `many=False` serialization (one object).
645
646    #### Usage with fields argument
647    All fields set on fields argument will be returned dispite beeing
648    foreign key or related and arguments `related_fields=False`,
649    `foreign_key_fields=False`.
650
651    Examples:
652        ```python
653        from serializers import SerializerCompany
654
655
656        # Defining serializer for User object
657        class SerializerUser(DynamicFieldsModelSerializer):
658            pk = serializers.IntegerField(source='id', allow_null=True)
659            model_class = ClassNameField()
660
661            # ForeignKey
662            api_permission_set = LocalRelatedField(
663                serializer=(
664                    "pumpwood_djangoauth.api_permission."
665                    "serializers.SerializerPumpwoodPermissionPolicyUserM2M"),
666                order_by=["-id"])
667            api_permission_group_set = LocalRelatedField(
668                serializer=(
669                    "pumpwood_djangoauth.api_permission."
670                    "serializers.SerializerPumpwoodPermissionUserGroupM2M"),
671                order_by=["-id"])
672            company = LocalRelatedField(serializer=SerializerCompany)
673
674            class Meta:
675                model = User
676                fields = (
677                    'pk', 'model_class', 'username', 'email', 'first_name',
678                    'last_name', 'last_login', 'date_joined', 'is_active',
679                    'is_staff', 'is_superuser', 'mfa_method_set',
680                    'api_permission_set', 'api_permission_group_set',
681                    'company')
682                list_fields = [
683                    "pk", "model_class", 'is_active', 'is_superuser',
684                    'is_staff', 'username', 'email', 'last_login']
685                read_only = ('last_login', 'date_joined')
686
687
688        # Query for User objects
689        all_users = User.objects.all()
690
691        # Create a serializer that will return just ['pk', 'username']
692        # fields when dumping the objects.
693        user_data = UserDynamicFieldsModelSerializer(
694            all_users, many=True, fields=['pk', 'description'],
695            foreign_key_fields=False, related_fields=False,
696            default_fields=False)
697
698        # Return all fields except for foreign key and related models.

A ModelSerializer that change fields returned on serialization.

This serializer make possible to change the serialization fields acording to arguments that are passed on serializer instanciation.

Default list fields

Attribute list_fields set the default fields that will be returned if default_fields=True argument is set.

Usage with foreign key

Foreign keys can be returned both for many=True and also many=False serializations passing foreign_key_fields=True argument.

Usage with related objects

Serialization of many objects will not return related models serializers ( MicroserviceRelatedField, LocalRelatedField). This behaviour will overide related_fields argumment, even if set as related_fields=True.

Related Models serizalization is an expensive request at backend, should be used only for many=False serialization (one object).

Usage with fields argument

All fields set on fields argument will be returned dispite beeing foreign key or related and arguments related_fields=False, foreign_key_fields=False.
